# Skill Quality Auditor Report Schema

This document defines the JSON output contract generated by `scripts/audit_skills.py`.

## Root Structure

```json
{
  "meta": {
    "skills_root": "string",
    "rubric_version": "1.0.0",
    "generated_at": "ISO-8601"
  },
  "summary": {
    "total_skills": 0,
    "average_score": 0,
    "by_band": {
      "good": 0,
      "fair": 0,
      "poor": 0
    }
  },
  "skills": []
}
```

## `skills[]` Item

```json
{
  "skill_name": "string",
  "skill_path": "absolute or relative path",
  "total_score": 0,
  "max_score": 100,
  "quality_band": "poor|fair|good",
  "criteria": {
    "metadata_frontmatter": {
      "score": 0,
      "max_score": 20,
      "status": "good|attention|critical",
      "evidence": ["string"]
    }
  },
  "critical_issues": ["criterion_key"],
  "recommendations": ["string"]
}
```

## Notes

- `quality_band` thresholds:
  - `poor` < 60
  - `fair` 60-79
  - `good` >= 80
- `status` is per-criterion and derived from score ratio:
  - `good` >= 80%
  - `attention` 50%-79%
  - `critical` < 50%
