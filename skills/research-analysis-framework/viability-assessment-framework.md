# Viability Assessment Framework - 6 Dimensions

## Purpose

Evaluate proposed projects/decisions across six independent dimensions. Each dimension gets its own 0-10 score. No single score is sufficient; all six must be healthy for a project to be truly viable.

---

## The Six Dimensions

### 1. TECHNICAL FEASIBILITY

**Question**: Can it be done from a technical/engineering standpoint?

#### What to Assess

| Area | Questions to Ask | Red Flags |
|------|---|---|
| **Specifications** | Do specs conform to standards/codes? | Non-standard, novel, unproven technology |
| **Feasibility** | Is it technically possible? | Impossible tolerances, conflicting requirements |
| **Standards Compliance** | Does it meet industry standards? | Code violations, non-compliance |
| **Resource Requirements** | Do we have the skills/tools needed? | Specialized skills not available |
| **Timeline Realism** | Can this be done in the timeframe? | Compressed timeline, parallel dependencies |
| **Proven Methods** | Has this been done before successfully? | First-of-a-kind, novel approach |

#### Scoring Guidance

```
9-10: Clearly feasible; proven methods; all specs compliant
7-8: Feasible; standard approach; minor complexity
5-6: Feasible but requires attention; some technical challenges
3-4: Significant technical barriers; novel approach; specialized skills needed
1-2: Questionable feasibility; high technical risk
0: Impossible; violates physical laws or core standards
```

#### Example Analysis

**House Foundation on Slope**
- STANDARD: Foundation on slopes is common practice (✓ proven)
- SPECS: Code requires specific soil testing & design (✓ compliant)
- SKILLS: Structural engineer needed (✓ available)
- TIMELINE: 4 weeks is standard (✓ realistic)
- **Score: 8/10** (technically solid, standard approach)

### 2. FINANCIAL VIABILITY

**Question**: Can we afford it? What's the return on investment?

#### What to Assess

| Area | Questions to Ask | Red Flags |
|------|---|---|
| **Total Cost** | What's the realistic project cost? | Estimates too low, missing contingency |
| **Funding** | How will this be funded? Cash/loan/other? | No clear funding source |
| **Budget Variance** | Cost vs. budget available? | Over budget, tight margins |
| **ROI/Payback** | What's the return? When do we break even? | Negative ROI, long payback, no clear benefit |
| **Cash Flow** | When is money needed/returned? | Large upfront, long tail on returns |
| **Contingency** | What buffer exists for overruns? | No contingency, tight budget |
| **Comparable Projects** | How does this compare to similar projects? | Significantly cheaper/more expensive (why?) |

#### Scoring Guidance

```
9-10: Costs clear & reasonable; strong ROI; good contingency; well-funded
7-8: Costs understood; acceptable ROI; moderate contingency
5-6: Costs unclear in places; ROI exists but modest; tight budget
3-4: Significant cost uncertainty; weak ROI; inadequate contingency
1-2: Costs likely underestimated; poor ROI; no contingency
0: Unaffordable; negative ROI; unfundable
```

#### Example Analysis

**House Construction Budget**
- TOTAL COST: Benchmarks suggest R$ 500k-600k (estimate: R$ 550k) ✓
- BUDGET: Have R$ 500k == tight (⚠)
- CONTINGENCY: 0% contingency in budget (🔴 HIGH RISK)
- ROI: Not applicable (primary residence)
- **Score: 5/10** (financially feasible but tight; no contingency)

### 3. REGULATORY COMPLIANCE

**Question**: Does it meet legal/regulatory requirements?

#### What to Assess

| Area | Questions to Ask | Red Flags |
|------|---|---|
| **Permits Required** | What permits/approvals needed? | Missing permits, approval timeline unclear |
| **Codes/Standards** | Does it comply with applicable codes? | Code violations, non-compliant design |
| **Licensing** | Do participants have proper licenses? | Unlicensed contractors, expired credentials |
| **Insurance** | Required insurance in place? | Inadequate coverage, missing policies |
| **Timeline** | How long will approvals take? | Delayed permits, extended approval process |
| **Cost** | Are there permit/compliance costs? | Underestimated regulatory costs |
| **Enforcement Risk** | What's risk if non-compliant? | High enforcement risk, penalties possible |

#### Scoring Guidance

```
9-10: Full compliance; all permits clear; quick approval; minimal risk
7-8: Compliant; standard permits; reasonable timeline
5-6: Largely compliant; some permit uncertainties; moderate timeline
3-4: Significant compliance gaps; permit delays possible; some risk
1-2: Major non-compliance; serious permit hurdles; high enforcement risk
0: Illegal; cannot be permitted; unachievable legally
```

#### Example Analysis

**House Construction Permits**
- BUILDING PERMIT: Required, standard process (3-4 weeks) ✓
- ZONING: Property zoned residential, complies ✓
- ELECTRICAL/PLUMBING: Licensed contractors required ✓
- FINAL INSPECTION: Required before occupancy ✓
- **Score: 9/10** (completely compliant; standard permitting)

### 4. MARKET VIABILITY

**Question**: Is there demand? Will it succeed in the market?

#### What to Assess

| Area | Questions to Ask | Red Flags |
|------|---|---|
| **Market Size** | How big is potential market? | Niche market, limited demand |
| **Demand Signals** | Is there proven demand? | New market, unproven demand |
| **Competition** | Who are competitors? What's market position? | Crowded market, strong competitors |
| **Pricing Power** | Can we price appropriately? | Price-sensitive market, commoditized |
| **Growth/Decline** | Is market growing or shrinking? | Declining market, poor outlook |
| **Adoption Timeline** | How fast will market adopt? | Slow adoption, long payback |
| **Risk of Disruption** | Could technology/market disrupt? | Disruptive technology emerging, market risk |

#### Scoring Guidance

```
9-10: Large market; strong demand; competitive advantage; growth trajectory
7-8: Solid market; evident demand; defensible position
5-6: Viable market; some demand; moderate competition
3-4: Niche market; limited demand; competitive pressure
1-2: Small market; weak demand; strong competition; declining
0: No market; no demand; doomed to fail
```

#### Example Analysis

**House Construction in São Paulo**
- MARKET: High demand for housing in metro area ✓
- DEMAND: Demographic trends support continued demand ✓
- COMPETITION: Many developers (but not oversupply) ✓
- PRICING: Market supports R$ 3500-4000/m² in this area ✓
- **Score: 9/10** (strong market, proven demand, no disruption risk)

### 5. OPERATIONAL FEASIBILITY

**Question**: Can we actually execute and deliver this?

#### What to Assess

| Area | Questions to Ask | Red Flags |
|------|---|---|
| **Resources** | Do we have people, tools, infrastructure? | Key skills missing, understaffed |
| **Skill Gaps** | Will we need to hire/train? | New skills needed, hiring challenges |
| **Supply Chain** | Can we source required materials/services? | Supply bottlenecks, long lead times |
| **Timeline** | Is schedule realistic given resources? | Compressed timeline, resource conflicts |
| **Dependencies** | What needs to happen first? | Uncontrollable dependencies, sequence risks |
| **Key Person Risk** | Does success depend on one person? | Single point of failure, key person at risk |
| **Scalability** | Can we handle growth if needed? | Limited capacity for growth |

#### Scoring Guidance

```
9-10: Clear execution path; necessary resources available; no bottlenecks
7-8: Executable with standard team; some hiring; manageable dependencies
5-6: Executable but requires attention; skill gaps exist; some constraints
3-4: Significant execution challenges; major hiring/training needed; bottlenecks
1-2: Severe execution risk; critical skills unavailable; major dependencies
0: Cannot execute; missing critical resources; impossible timeline
```

#### Example Analysis

**House Construction Execution**
- CONTRACTOR: Qualified, experienced (✓)
- CREW: Available labor in market (✓)
- MATERIALS: Standard materials, available suppliers (✓)
- TIMELINE: 12 months is standard for this size (✓)
- DEPENDENCIES: Permits (external but manageable) (✓)
- **Score: 8/10** (clear execution path, standard process, few bottlenecks)

### 6. RISK PROFILE

**Question**: What could go wrong? What's our mitigation?

#### What to Assess

| Area | Questions to Ask | Red Flags |
|------|---|---|
| **Risk Inventory** | What could fail? What's risky? | Many unknowns, high-impact risks |
| **Probability** | How likely are key risks? | High-probability risks, common failures |
| **Impact** | If it happens, how bad? | High-impact failures, project-ending risks |
| **Controllability** | Can we control/mitigate risks? | Uncontrollable risks, external dependencies |
| **Mitigation** | What's our plan if risk occurs? | No mitigation, no contingencies |
| **Contingency Budget** | Do we have buffer/reserves? | No contingency, tight budget |
| **Insurance** | What's insurable/protected? | Risks uninsurable, unprotected exposure |

#### Risk Scoring Framework

```
Risk Matrix:
         Low Impact    Medium Impact    High Impact
Low Prob    ✓              ✓              ✓ (monitor)
Med Prob    ✓              ⚠ (mitigate)   🔴 (deal with)
High Prob   ⚠ (mitigate)   🔴 (deal with) 🔴 (blocker)
```

#### Scoring Guidance

```
9-10: Minimal risks; well-mitigated; good contingency; low probability issues
7-8: Known risks; adequate mitigation; reasonable contingency
5-6: Moderate risks; some mitigation gaps; tight contingency
3-4: Significant risks; limited mitigation; at-risk contingency
1-2: Major risks; inadequate mitigation; no contingency
0: Unacceptable risk; unmitigable failure modes; project doomed
```

#### Example Analysis

**House Construction Risks**
- WEATHER: Common risk; 3-month contingency in schedule (✓ mitigated)
- COST OVERRUNS: Material costs could rise; 15% contingency (⚠ tight)
- CONTRACTOR BANKRUPTCY: Unlikely with established firm; insurance covers (✓)
- PERMIT DELAYS: Possible; have 1-month buffer (✓ mitigated)
- **Primary Risk**: Budget is tight (0% contingency is HIGH RISK)
- **Score: 5/10** (known risks but tight contingency buffers)

## Composite Viability Score

### Calculation

**Simple Average**:
```
Viability Score = (Technical + Financial + Regulatory + Market + Operational + Risk) / 6
```

**Example** (House Construction):
```
Technical:    8/10
Financial:    5/10  ← tight budget
Regulatory:   9/10
Market:       9/10
Operational:  8/10
Risk:         5/10  ← high contingency risk
──────────────────
AVERAGE:      7.3/10  → "GOOD but with concerns"
```

### Interpretation

```
9-10: EXCELLENT - Proceed with confidence
7-8: GOOD - Clear path forward; manage identified risks
5-6: MODERATE - Viable but requires attention; manage gaps
3-4: WEAK - Significant barriers; requires major mitigation
1-2: POOR - Questionable viability; extensive changes needed
0: BLOCKER - Cannot proceed in current form
```

### Weighted Scoring (Alternative)

If some dimensions matter more:

```
Technical:     8/10 × 15% = 1.20
Financial:     5/10 × 30% = 1.50  ← More important for this decision
Regulatory:    9/10 × 20% = 1.80
Market:        9/10 × 15% = 1.35
Operational:   8/10 × 15% = 1.20
Risk:          5/10 × 5%  = 0.25
──────────────────────────────────
WEIGHTED SCORE = 7.30/10
```

## Quality Checklist

For each dimension scoring:

- [ ] Evidence from sources (tier-1 preferred)
- [ ] Red flags explicitly identified
- [ ] Confidence level assigned (🟢🟡🟠🔴)
- [ ] Mitigations documented (if risk identified)
- [ ] Realistic relative to benchmarks
- [ ] No magical thinking (inflated scores)
- [ ] All gaps acknowledged
- [ ] Appropriate caveats noted
