# Presentation Guidelines - Adapting for Different Audiences

## Purpose

Same analysis, different audiences. This guide shows how to adapt findings for executive vs technical vs finance teams. Key: adjust detail level and framing, not conclusions.

---

## Core Principle

Different stakeholders need different information:

```
EXECUTIVE LEADER
├─ Wants: "What should we decide?" + "What's the risk?"
├─ Detail Level: High-level summary
├─ Format: Deck or brief
├─ Time: 5-15 minutes
└─ Goal: Enable decision

TECHNICAL TEAM
├─ Wants: "How do we build/execute this?" + "What could go wrong?"
├─ Detail Level: Comprehensive technical depth
├─ Format: Full report + discussion
├─ Time: 1-2 hours
└─ Goal: Enable implementation

FINANCE TEAM
├─ Wants: "What does this cost?" + "What's ROI?" + "What are scenarios?"
├─ Detail Level: Cost modeling detail
├─ Format: Spreadsheets + explanation
├─ Time: Detailed modeling discussion
└─ Goal: Enable financial planning

PROJECT MANAGER
├─ Wants: "What's the timeline?" + "What are dependencies?" + "Resources needed?"
├─ Detail Level: Execution + sequencing
├─ Format: Gantt, milestones, resource plan
├─ Time: Work session planning
└─ Goal: Enable execution planning
```

## Format by Audience

### 1. EXECUTIVE BRIEF (CEO/Board/Investor)

**Duration**: 10-15 minutes
**Format**: 1-page written brief + optional 15-min discussion
**Tone**: Professional, confident, decision-focused

#### Content Structure

```
EXECUTIVE BRIEF: Housing Investment Decision
──────────────────────────────────────────────

THE DECISION
[What question are we answering?]
"Should we proceed with R$ 550k residential construction investment?"

THE RECOMMENDATION
[Your answer - clearly stated]
"PROCEED if budget is increased to R$ 575k with 15% contingency"

VIABILITY SCORE
[One number - 0-10 or STOP/CAUTION/GO]
7.3/10 (GOOD - proceed with risk management)

KEY FINDINGS (3-5 bullets)
├─ Market opportunity: Strong demand in target area (🟢 HIGH confidence)
├─ Technical feasibility: Standard residential construction (🟢 HIGH confidence)
├─ Financial viability: Affordable but requires contingency (🟡 MEDIUM concern)
├─ Regulatory path: Clear permitting process (🟢 HIGH confidence)
└─ Execution risk: Depends on contractor quality (🟡 MEDIUM)

TOP 3 RISKS
├─ #1 (CRITICAL): Budget has zero contingency → Add 15% buffer
├─ #2 (HIGH): Material cost inflation not protected → Lock quotes early
└─ #3 (MEDIUM): Contractor change mid-project → Would be expensive

FINANCIAL SUMMARY
├─ Investment required: R$ 575k (approved + contingency)
├─ Expected value: R$ 750k asset value (residential market)
├─ Net position: +R$ 175k equity (if no cost overruns)
├─ ROI: Not applicable (primary residence)
└─ Payback: 7-year hold (equity accumulation model)

NEXT STEPS
├─ 1. Approve R$ 575k budget [DECISION REQUIRED THIS WEEK]
├─ 2. Complete soil testing [Q1 2024]
├─ 3. Submit permits [Q1 2024]
└─ 4. Confirm contractor [Q1 2024]

TIMELINE
├─ Decision deadline: March 15, 2024
├─ Construction start: April 2024 (subject to permits)
├─ Completion: April 2025 (12-month timeline)
└─ Ready for occupancy: May 2025

For details, see full analysis attached.
Questions? Contact [analyst contact]
```

**Presentation Tips**:
- Lead with recommendation
- Use 1 slide per section
- Highlight top 3 risks visually
- Show decision timeline
- Ask for specific approval

### 2. DIRECTOR/VP PRESENTATION (8-10 slides)

**Duration**: 30-45 minutes
**Format**: PowerPoint deck + Q&A
**Tone**: Professional, detailed enough for decision, visual

#### Slide Deck Structure

```
Slide 1: TITLE & CONTEXT
├─ Topic: Housing Investment Analysis
├─ Stakeholders: [Who's deciding?]
├─ Timeline: [Decision needed by when?]
└─ Recommendation: PROCEED WITH CONDITIONS

Slide 2: THE QUESTION
├─ What are we deciding?
├─ Why is this important?
├─ What are the consequences?
└─ [1 visual: decision tree or challenge]

Slide 3: EXECUTIVE SUMMARY
├─ Viability score: 7.3/10 (GOOD)
├─ Recommendation: PROCEED IF budget increased to 575k
├─ Key conditions: Add contingency, lock supplier quotes
└─ [1 visual: summary scorecard]

Slide 4: OPPORTUNITY
├─ Market demand in area
├─ Competitive position
├─ Expected value creation
└─ [1 visual: market data chart]

Slide 5: ASSESSMENT OVERVIEW
├─ Six dimensions evaluated
├─ Scoring methodology
├─ What this decision depends on
└─ [1 visual: 6-dimension scorecard]

Slide 6: TECHNICAL & REGULATORY
├─ Technical feasibility: 8/10 (standard construction)
├─ Regulatory path: 9/10 (clear permits)
├─ Compliance: Confirmed with authorities
└─ [1 visual: compliance checklist or timeline]

Slide 7: FINANCIAL PICTURE
├─ Cost estimate: 550k (vs. 500k budget = issue)
├─ Recommended budget: 575k (includes 15% contingency)
├─ ROI/Value: 750k expected value = positive
├─ Risks: Cost inflation, contingency inadequacy
└─ [1 visual: cost breakdown + sensitivity table]

Slide 8: RISKS & MITIGATIONS
├─ Risk 1: Budget tightness → Mitigation: Add contingency
├─ Risk 2: Timeline pressure → Mitigation: Experienced contractor
├─ Risk 3: Cost creep → Mitigation: Lock quotes, contingency
└─ [1 visual: risk matrix (probability × impact)]

Slide 9: RECOMMENDATION & NEXT STEPS
├─ DECISION: Approve 575k budget
├─ ACTION 1: [What, who, when]
├─ ACTION 2: [What, who, when]
├─ ACTION 3: [What, who, when]
└─ [1 visual: timeline + owners]

Slide 10: Q&A
└─ [Prepare for questions on: budget details, contractor, timeline, risk]
```

**Presentation Tips**:
- One idea per slide
- Use visuals (charts > tables > text)
- Avoid walls of text
- Highlight top 3 risks
- Prepare for tough questions
- Be comfortable with confidence levels (don't oversell)

### 3. TECHNICAL TEAM BRIEFING (Full Report + Presentation)

**Duration**: 2-3 hours
**Format**: Full 15-page analysis report + 2-hour work session
**Tone**: Detailed, technical, execution-focused

#### Content Structure

```
SECTION 1: TECHNICAL ANALYSIS (30 min)
├─ Design specifications overview
├─ Code compliance assessment
├─ Material specifications
├─ Construction methodology
├─ Supplier/contractor assessment
└─ "Can we build this?" → Answer: Yes, standard process

SECTION 2: EXECUTION PLAN (45 min)
├─ Phase 1: Foundation (timeline, resources, risks)
├─ Phase 2: Structure (timeline, resources, risks)
├─ Phase 3: MEP (timeline, resources, risks)
├─ Phase 4: Finishes (timeline, resources, risks)
├─ Critical path & dependencies
└─ "What does it take to execute?" → Answer: [detailed plan]

SECTION 3: QUALITY & COMPLIANCE (30 min)
├─ Quality standards & inspections
├─ Code compliance verification
├─ Testing & certification requirements
├─ Punch list & final sign-off
└─ "How do we guarantee quality?" → Answer: [QA/QC plan]

SECTION 4: RISK & MITIGATION (15 min)
├─ Schedule risks (weather, dependencies, contractor)
├─ Quality risks (material, workmanship, inspection)
├─ Safety risks (standard precautions, insurance)
├─ Escalation path (who decides if issues arise?)
└─ "What could go wrong?" → Answer: [mitigation plan]

Work Session: DEEP DIVES
├─ Technical Q&A on subsystems
├─ Detailed review of contractor proposal
├─ Timeline pressure-testing
├─ Resource adequacy discussion
└─ "What aren't we thinking about?" → Group problem-solving
```

**Presentation Tips**:
- Assumes technical background
- Dive into specifications
- Discuss trade-offs and alternatives
- Invite challenge and questioning
- Document decisions in writing
- Establish escalation process

### 4. FINANCE TEAM PRESENTATION (Modeling + Discussion)

**Duration**: 1-2 hours
**Format**: Cost model spreadsheet + sensitivity analysis
**Tone**: Detailed, quantitative, scenario-focused

#### Content Structure

```
SECTION 1: COST MODEL (30 min)
├─ Cost estimate breakdown
│   ├─ Foundation: R$ 80k
│   ├─ Structure: R$ 200k
│   ├─ MEP: R$ 120k
│   ├─ Finishes: R$ 130k
│   └─ Contingency: R$ 25k (15% - RECOMMENDED)
├─ Comparison to budget
├─ Benchmark comparison (industry rates)
└─ Confidence: 75% (subject to validation)

SECTION 2: FUNDING & CASH FLOW (20 min)
├─ Total investment: R$ 575k
├─ Funding source: [equity/loan/other]
├─ Payment schedule: [What contractor gets when?]
├─ Cash flow impact: [Monthly drawdown]
├─ Working capital needs: [Contingency access?]
└─ Questions: "Can we fund this?" Answer: [Yes/No/Conditional]

SECTION 3: FINANCIAL SCENARIOS (30 min)
├─ SCENARIO A (Base case): R$ 550k cost, on-time
│   └─ End value: R$ 750k (per market assessment)
│ 
├─ SCENARIO B (Optimistic): R$ 520k cost, early completion
│   └─ End value: R$ 750k (no time premium expected)
│
├─ SCENARIO C (Pessimistic): R$ 600k cost, late completion
│   └─ End value: R$ 725k (minor market adjustment)
│
└─ Sensitivity Analysis
    ├─ If cost +5% → Budget impact: R$ 27.5k (manageable with contingency)
    ├─ If cost +15% → Budget impact: R$ 82.5k (BREAKS contingency - escalate)
    ├─ If delay 6 months → Market impact: TBD (requires reassessment)
    └─ If market price falls 10% → Value: R$ 675k (still positive)

SECTION 4: FINANCIAL COMPARISON (10 min)
├─ Option A: Build (R$ 575k investment, 7-year hold)
├─ Option B: Rent (R$ 2k/month = R$ 168k over 7 years)
├─ Option C: Don't invest (avoid R$ 575k, lose upside)
└─ Financial advantage: Building wins over renting (based on market appreciation)
```

**Finance-Specific Metrics**:

```
Investment Metrics
├─ Initial investment: R$ 575k
├─ Expected value: R$ 750k (after 7 years)
├─ Net gain: R$ 175k
├─ ROI: 30% cumulative (4.3% annualized)
├─ Payback period: 6-7 years (through market appreciation)
└─ Risk: Market downturns would reduce returns

Cost Control Metrics
├─ Cost estimate confidence: 75% (±10% range: R$ 495-605k)
├─ Contingency level: 15% (R$ 25k available)
├─ Budget flexibility: Low (tight approval limit)
├─ Escalation trigger: If costs exceed R$ 560k
└─ Overrun authority: Finance approval required >R$ 575k

Cash Flow Metrics
├─ Month 1-3: Foundation work → R$ 80k spent
├─ Month 4-9: Structure work → R$ 200k spent
├─ Month 10-13: Finishes → R$ 215k spent
├─ Reserve: R$ 25k contingency retained
└─ Total outlay: R$ 575k over 13 months
```

**Presentation Tips**:
- Provide detailed cost model
- Show 3-5 scenarios
- Use sensitivity analysis
- Highlight contingency assumptions
- Discuss funding logistics
- Address ROI/value explicitly

## Audience Adaptation Framework

### For Each Audience, Answer

```
1. What does this person care about most?
   Executive: Decision confidence, risk, timeline
   Technical: Feasibility, specifications, execution
   Finance: Cost, ROI, contingency, funding
   PM: Timeline, dependencies, resources

2. What detail level is appropriate?
   Executive: Summary + top 3 risks
   Technical: Full technical specifications
   Finance: Cost model + sensitivities
   PM: Gantt, milestones, resource plan

3. What format works best?
   Executive: 1-page brief or 10-min deck
   Technical: Full report + work session
   Finance: Model + presentation
   PM: Timeline + resource plan

4. What questions will they ask?
   Executive: "Why should we do this?" "What if it breaks?"
   Technical: "Is this feasible?" "What are specs?" "How do we QA?"
   Finance: "What does it cost?" "What's the ROI?" "What if costs rise?"
   PM: "When?" "How many people?" "What depends on what?"

5. What decision does this briefing enable?
   Executive: Approval/rejection decision
   Technical: Design/implementation approach
   Finance: Funding authorization
   PM: Execution planning + sequencing
```

## Presentation Standards (All Formats)

✅ **DO**:
- Start with recommendation
- Show confidence levels (🟢🟡🟠🔴)
- Highlight top 3 risks
- Make it skimmable
- Use visuals when possible
- Explain jargon
- Provide source citations
- Answer "So what?" for every finding

❌ **DON'T**:
- Assume audience knows context
- Bury key findings in detail
- Claim certainty you don't have
- Hide contradictions
- Use dense tables (charts better)
- Talk for more than 50% of time (invite Q&A)
- Oversimplify technical topics
- Make unsupported recommendations
