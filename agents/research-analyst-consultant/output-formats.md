# Output Formats - research-analyst-consultant

## Standard Deliverables

The agent produces multiple formats depending on needs. Users typically receive:

1. **Written Report** (PDF or Markdown)
2. **Executive Presentation** (Slides)
3. **Data Pack** (Tables, spreadsheets, detailed findings)
4. **Brief** (1-2 page decision summary)

---

## 1. WRITTEN REPORT

### Standard Structure

```
REPORT TITLE: [Topic] - Research & Viability Analysis

EXECUTIVE SUMMARY (1 page)
├─ Question/Objective
├─ Key Finding
├─ Recommendation
├─ Viability Score (if applicable)
├─ Top 3 Risks
└─ Next Steps

DETAILED FINDINGS (8-15 pages)
├─ Analysis Framework
├─ Dimension 1: Technical
│   ├─ Assessment approach
│   ├─ Source-based evidence
│   ├─ Key findings
│   ├─ Confidence level (🟢🟡🟠🔴)
│   └─ Risks identified
├─ Dimension 2: Financial
│   [... same structure ...]
├─ Dimension 3: Regulatory
│   [... same structure ...]
├─ Dimension 4: Market
│   [... same structure ...]
├─ Dimension 5: Operational
│   [... same structure ...]
└─ Dimension 6: Risk Profile
    [... same structure ...]

EVIDENCE & SOURCES (3-5 pages)
├─ Source Inventory
│   ├─ Which sources were used?
│   ├─ Gaps in coverage
│   └─ Credibility assessment
├─ Chain of Reasoning
│   ├─ How findings connect
│   ├─ Evidence hierarchy
│   └─ Assumptions underlying analysis
├─ Contradictions Identified
│   ├─ Where sources disagree
│   ├─ Credibility assessment
│   └─ Resolution approach
└─ Confidence Matrix
    └─ By finding (high/medium/low/unverified)

RECOMMENDATIONS & NEXT STEPS (2-3 pages)
├─ Priority Actions
│   ├─ What to do first
│   ├─ Timeline
│   └─ Owner/Responsibility
├─ Decision Criteria
│   ├─ What information would change recommendation?
│   └─ Go/no-go thresholds
├─ Contingency Plans
│   ├─ If key assumption breaks
│   ├─ Backup options
│   └─ Risk mitigation
└─ Monitoring & Evolution
    ├─ How will we track progress?
    ├─ When should we re-assess?
    └─ What new data would improve analysis?

APPENDICES
├─ Detailed Data Tables
├─ Source Summaries
├─ Calculation Workings
├─ Methodology Notes
└─ Assumptions List
```

### Report Characteristics

- **Length**: 12-20 pages (exec summary + findings + evidence + recommendations)
- **Format**: PDF, professionally formatted with headers, tables, charts
- **Visuals**: Charts, graphs, matrices where data supports
- **Language**: Clear, accessible (avoid jargon; explain technical terms)
- **Traceability**: Every claim tied to source
- **Confidence**: Marked on key findings
- **Audience**: Middle managers who want depth but not overwhelming detail

## 2. EXECUTIVE PRESENTATION

### For C-Level Leadership

**Format**: PowerPoint or PDF slides (8-10 slides)

```
SLIDE 1: TITLE SLIDE
├─ Question/Objective
├─ Date
└─ Prepared for: [Audience]

SLIDE 2: RECOMMENDATION
├─ Recommended Action
├─ Viability Score
└─ Timeline to Decision

SLIDE 3-4: TOP FINDINGS (Technical, Financial, Regulatory)
├─ 1-2 key facts per slide
├─ Visuals (charts > text)
└─ Confidence level

SLIDE 5: TOP 3 RISKS
├─ Risk #1: Description + Mitigation
├─ Risk #2: Description + Mitigation
└─ Risk #3: Description + Mitigation

SLIDE 6: NEXT STEPS
├─ What to do
├─ Timeline
└─ Owner

SLIDE 7: KEY ASSUMPTIONS
├─ Assumptions underlying recommendation
├─ Which are validated vs. risky?
└─ What could change recommendation?

SLIDE 8: QUESTIONS & DECISION
├─ What other info would help?
├─ When do we need to decide?
└─ How will we monitor?
```

**Characteristics**:
- **Visual**: 70% charts/graphics, 30% text
- **Concise**: 5-7 min to present
- **Action-oriented**: Reader knows what to do
- **Risk-aware**: Top risks visible
- **Confidence**: Stated in findings

## 3. DATA PACK

For analysts and technical teams who want full detail:

### Contents

```
DATA PACK: [Topic] - Supporting Detail

[File 1] source-inventory.xlsx
├─ All sources used
├─ Credibility tier
├─ Key topics covered
├─ Limitations noted
└─ Cross-references

[File 2] evidence-index.md
├─ Evidence by dimension
├─ Source attribution
├─ Quote/citation details
├─ Confidence assessment
└─ Contradictions flagged

[File 3] viability-scoring-matrix.xlsx
├─ Dimension-by-dimension scores (0-10)
├─ Scoring justification
├─ Key metrics by dimension
└─ Comparison to benchmarks

[File 4] financial-model.xlsx
├─ Cost-benefit analysis
├─ Sensitivity analysis
├─ Scenario modeling (best/likely/worst case)
└─ ROI calculations

[File 5] risk-register.xlsx
├─ Identified risks
├─ Probability & impact
├─ Mitigation strategies
└─ Contingency budget

[File 6] contradiction-analysis.md
├─ Where sources disagree
├─ Source credibility assessment
├─ Resolution approach
└─ Data quality notes

[File 7] assumptions-list.md
├─ Every assumption in analysis
├─ Which are validated vs. at-risk
├─ Impact if assumption breaks
└─ Monitoring approach
```

**For**: Technical teams, finance teams, detailed analysts
**Detail Level**: Everything (the full evidence base)
**Format**: Spreadsheets + markdown

## 4. EXECUTIVE BRIEF

### For Time-Constrained Decision-Makers

**Format**: 1-2 pages (Markdown or PDF)

```
EXECUTIVE BRIEF

TO: [Decision-Maker Name]
FROM: [Analyst Name]
DATE: [Date]
RE: [Decision Required] - RECOMMENDATION & SUMMARY

BOTTOM LINE
[One sentence: recommended decision]

RECOMMENDATION
[3-5 sentences: what to do and why]

KEY FACTS
├─ Technical: [1-2 facts]
├─ Financial: [1-2 facts]
├─ Regulatory: [1-2 facts]
├─ Market: [any critical info]
└─ Risk: [Top risk + mitigation]

VIABILITY SCORE
[0-10 score + brief justification]
Scoring basis: Technical✓ Financial✓ Regulatory✓ Market✓ Operational✓ Risk✓

TIMELINE
Next decision: [When needed?]
Implementation starts: [If approved]
Full completion: [Timeline]

CONFIDENCE LEVEL
Overall finding confidence: 🟢 HIGH / 🟡 MEDIUM / 🟠 LOW
Main uncertainty: [What are we least sure about?]

WHAT COULD CHANGE THIS RECOMMENDATION
- [Condition 1 that would reverse recommendation]
- [Condition 2 that would change scope]
- [Condition 3 that would affect timeline]

QUESTIONS?
[Contact info for detailed discussions]

---
For details, see: [Full Report Link]
For data, see: [Data Pack Link]
```

**Characteristics**:
- **5 min read**: Decision-maker can absorb in brief meeting
- **Decision-ready**: Clear recommendation
- **Risk-aware**: Key risks flagged
- **Evidence**: Confidence levels shown

## Format Selection Guide

| Decision Maker | Primary Format | Secondary |
|---|---|---|
| CEO/CFO (1 page) | Executive Brief | Presentation slides |
| VP/Director (5 min prep) | Presentation Slides | Brief |
| Technical Lead (deep dive) | Data Pack | Full Report |
| Project Manager (execution) | Implementation Roadmap | Risk Register |
| Finance Team (analysis) | Financial Model | Evidence Index |
| Board/Steering Committee | Presentation + Brief | Report Deck |

## Delivery Standards

All formats meet these criteria:

✅ **Traceability**: Claims → Evidence → Source (reader can verify)
✅ **Confidence**: Every key finding has confidence level
✅ **Visuals**: Charts & graphs replace tables where possible
✅ **Clarity**: Jargon explained; complex concepts simplified
✅ **Actionability**: Clear next steps / decision path
✅ **Completeness**: All dimensions addressed
✅ **Professionalism**: Presentation-grade quality
✅ **Accessibility**: Multiple formats for different audiences

---

## Versioning & Evolution

Reports are **living documents**:

1. **Version 1.0**: Initial analysis with available sources
2. **Version 1.1**: Updated when minor new source added
3. **Version 2.0**: Re-analysis when major sources added or assumptions change
4. **Updates**: Quarterly refreshes on financial/market data

Each version notes:
- What changed from prior version
- Why recommendation changed (if it did)
- New risks/opportunities identified
- Improved confidence levels (if applicable)

---

## Delivery Checklist

Before delivering, agent confirms:

- [ ] All claims are source-traceable
- [ ] Confidence levels appear on key findings
- [ ] Contradictions in sources are surfaced
- [ ] Viability scores cover all relevant dimensions
- [ ] Unknowns/gaps are documented
- [ ] Report is professionally formatted
- [ ] Recommendations flow logically from findings
- [ ] Executive can read brief in 5 minutes
- [ ] Analysts can access full evidence if desired
- [ ] Document is ready for stakeholder presentation
