# AI Context

Centralized AI context management — one source of truth for Copilot, Gemini, Claude, Cursor and more.

## 📁 Structure

```
./.ai-context/
├── rules/                  ← P0 — Applied to every response
│   ├── ALWAYS.md           ← Auto-generated by aictx sync (do not edit)
│   ├── agent-protocol.md   ← P0/P1/P2 enforcement, routing, activation
│   ├── ai-chat-response-discipline.md ← Response scope & token discipline
│   ├── checklist-protocol.md ← Pre-response gate, code echo ban
│   ├── code-standards.md   ← Clean Code, File Awareness, Task Tiers
│   └── socratic-gate.md    ← Ask vs act — clarification protocol
├── skills/                 ← P2 — Loaded on demand by agents
│   ├── README.md           ← Skills system guide
│   ├── intelligent-routing/ ← meta-skill: auto agent selection
│   ├── knowledge-integrator/ ← meta-skill: doc sync
│   ├── skill-extractor/    ← meta-skill: detect new skill opportunities
│   ├── study-notes-writer/ ← meta-skill: generate study notes
│   └── ... (42 total)
├── agents/                 ← P1 — Applied when selected by routing
│   └── ... (20 total)
├── workflows/              ← Invoked explicitly by user
│   └── ... (11 total)
└── scripts/
    ├── aictx               ← Main CLI (Python)
    ├── aictx.sh            ← Shell wrapper
    └── install.sh          ← First-time setup
```

## 🚀 Quick Start

### 1. Install the CLI

```bash
cd ./.ai-context/scripts
./install.sh
```

Adds `aictx` to your PATH and sets `AICTX_HOME`.

### 2. Sync resources

```bash
aictx sync
```

Scans all folders, regenerates `rules/ALWAYS.md` with the current agents/skills/workflows, and updates Copilot instructions.

### 3. Configure AI tools

```bash
aictx install --all         # Install all targets without prompts
aictx install               # Interactive — choose targets + resources
aictx install --force       # Overwrite existing links
```

### 4. Verify

```bash
aictx status                # Show link status for all targets
aictx doctor                # Validate integrity (broken refs, orphans)
```

## 📋 Commands

| Command | Description |
|---------|-------------|
| `aictx sync` | Scan resources, regenerate ALWAYS.md + update Copilot instructions |
| `aictx install` | Interactive link installation |
| `aictx install --all` | Install all targets and resources |
| `aictx install --force` | Overwrite existing links |
| `aictx status` | Show current link status |
| `aictx doctor` | Validate integrity: broken skill refs, orphans, broken symlinks |
| `aictx add skill <name>` | Scaffold a new skill folder with SKILL.md |
| `aictx add agent <name>` | Scaffold a new agent .md file |
| `aictx uninstall` | Remove all managed links |
| `aictx config` | Show configuration and PATH setup |

## 🏗️ Adding New Skills or Agents

```bash
# Create a new skill (scaffolds skills/<name>/SKILL.md)
aictx add skill my-skill --description "What this skill does"

# Create a new agent
aictx add agent my-agent --description "Agent role" --skills clean-code,my-skill

# After editing the files:
aictx sync
```

## 🧠 Rule Levels

| Level | Files | When Active |
|-------|-------|-------------|
| **P0** | `rules/*.md` | Always — every response, every session |
| **P1** | `agents/{name}.md` | When agent is selected by intelligent-routing |
| **P2** | `skills/{name}/SKILL.md` | When loaded by selected agent's `skills:` frontmatter |

## 🏷️ Meta-Skills

Skills tagged `tag: meta` in their frontmatter are always active (like P0 rules). Current meta-skills:

- `intelligent-routing` — auto-selects the right agent for each request
- `knowledge-integrator` — flags outdated docs after decisions
- `skill-extractor` — detects opportunities for new skills
- `study-notes-writer` — generates study notes after learning blocks

To mark a skill as always-active: add `tag: meta` to its `SKILL.md` frontmatter, then run `aictx sync`.

Edit skills/agents/rules → aictx sync → ALWAYS.md regenerated → all AI tools updated
