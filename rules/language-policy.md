# Language Policy

## Core Rule

| Context | Language |
|---------|----------|
| Chat responses (conversational messages to the user) | Portuguese |
| Any file created, updated, or generated | English |

---

## What This Covers

### Always in Portuguese
- Conversational replies in chat
- Questions asked to the user
- Clarifications and confirmations
- Explanations of what was done
- Error messages and warnings to the user

### Always in English
- File contents when creating new files (`.md`, `.ts`, `.py`, `.json`, etc.)
- File contents when editing existing files
- Skill files (`SKILL.md`)
- Rule files
- Agent files
- Documentation generated by request
- Code comments and inline documentation
- Commit messages
- Pull request titles and descriptions
- Any artifact written to disk

---

## Enforcement

- This rule applies regardless of which language the user writes in
- If a document already exists in Portuguese and is being updated, migrate the updated sections to English
- Mixed-language documents are not acceptable — the target language for any file on disk is always English
- Chat remains Portuguese throughout — do not switch to English in conversational responses
